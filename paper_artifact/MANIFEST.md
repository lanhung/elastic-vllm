# Manifest and provenance

## One-command paths

```bash
make sim       # regenerate E1–E10 from the Azure traces
make sim JOBS=8 # parallelize independent E8/E9 points
make figs      # regenerate all figures, including hardware measurements
make paper     # regenerate figures and compile the paper
make all       # simulation plus paper
```

## Code

| file | purpose |
|---|---|
| `src/workload.py` | Load Azure traces and compose request records into agent programs |
| `src/sim.py` | Calibrated cluster simulator with active KV, reclaimable prefix cache, partial LRU eviction, and drain window |
| `src/policies.py` | HPA, KEDA, KV-util, static, ParkAware, and pressure-aware admission policies |
| `src/rl_policy.py` | Holt forecaster and tabular Q-learning baselines |
| `src/run_experiments.py` | E1–E6 |
| `src/exp7_sensitivity.py` | E7 sensitivity sweeps at T=8 and T=64 |
| `src/exp8_high_load.py` | E8 deterministic 1–8x load stress |
| `src/exp9_admission_sensitivity.py` | E9 P1-derived admission threshold sweep |
| `src/exp10_queue_tradeoff.py` | E10 queue/recompute/GPU accounting |
| `src/validate_results.py` | Canonical row-count, policy-completeness, and invariant checks |
| `src/make_figs.py` | Results and raw measurements to publication figures |
| `tests/test_sim.py` | Mechanism, accounting, calibration, stress-input, completeness, and headline regression tests |

## Input data

The two `data/AzureLLMInferenceTrace_*.csv` files are the Azure Public Dataset
traces used by the simulation. `data/get_data.sh` records their upstream URLs.
The repository versions used for the final run contain 8,819 coding records
and 19,366 conversation records.

## Hardware validation

`vllm_measured/summary.json` is the canonical numerical summary. Raw inputs are
under `vllm_measured/raw/`:

| file | experiment |
|---|---|
| `v1_prefix_cache.csv` | Cache hit/miss TTFT versus context length |
| `v2_parking_phases.csv`, `v2_parking_samples.csv` | Eight-session compute/park phases and Prometheus/DCGM samples |
| `v3_batch_curve.csv` | Concurrency 1–32 batch-throughput curve |
| `v4_cold_start.csv` | Three clean process cold starts |
| `p1_cache_survival.csv` | All 45 delay/pressure/repetition trials |
| `p1_cache_survival_summary.csv` | Aggregated P1 survival curve |
| `p2_pressure_admission.csv` | Protected and uncontrolled P2 trials |
| `p2_pressure_admission_summary.csv` | Aggregated P2 survival, TTFT, and goodput |

The validation scripts are under `vllm_validation/`. These measurements were
made on one machine and are not regenerated by `make sim`.

## Generated simulation outputs

| output | role |
|---|---|
| `results/e1_trace_stats.json` | Trace characterization |
| `results/e2_metric_divergence.csv`, `e2_timeline_T*.csv` | Native metric blindness and reclaimable cache state |
| `results/e3_policy_sweep.csv` | Main policy sweep and static reference |
| `results/e4_scale_in_damage.csv` | Token-level eviction/scale-in/recompute attribution |
| `results/e5_coldstart.csv` | Cold-start sensitivity including the 38.155 s measured point |
| `results/e6_workload_contrast.csv` | Coding versus conversation traces |
| `results/e7_sensitivity.csv` | KV capacity, max batch, and target sweeps |
| `results/e8_high_load.csv` | Synthetic load amplification and pressure-aware admission results |
| `results/e9_admission_sensitivity.csv` | Admission batches 4–16 at T=8/64 and 4–8x load |
| `results/e10_queue_tradeoff.csv` | HPA-relative queue, recomputation, GPU, SLO, and completion accounting |

`RESULTS_ANALYSIS.md` is the final human-readable numerical interpretation.

The final full run log is `logs_sim_final.txt`. `logs_sim_recalibrated.txt` and
`logs_e2_recalibrated.txt` are intermediate diagnostics retained to document
the correction process, not canonical outputs.

## Audit trail

`legacy_precalibration/` preserves the uploaded pre-measurement paper and
result bundle. The original simulator treated parked KV as reserved and visible
to KV-util; its headline results must not be mixed with the corrected outputs.
